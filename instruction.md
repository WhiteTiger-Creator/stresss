# Service-log export recovery

The service-log export deployed to the bridge is producing an unreliable flagged queue. Restore `/app/workflow/export_report.py` and build `/app/log_audit.py` with two subcommands: `diagnose --dossier PATH --report PATH` and `repair --output-dir PATH`. The diagnosis must cover all six known deployment defects — `wrong_timestamp_field`, `severity_filter`, `sort_order`, `level_normalization`, `dedupe_policy`, and `suppressed_filter` — each with verbatim dossier evidence and case-sensitive frozen-source excerpts; check every evidence field against `quick_contract_index.evidence_term_checklist` before finishing. A repair run writes `summary.json`, `service_matrix.json`, compact JSON-lines `flagged.jsonl`, `diagnosis.json`, and `repair_audit.json` under the requested output directory (`/app/output` by default), and a repair aimed at a custom directory must leave the default outputs untouched.

Events default to `/app/data/events.json` and the workflow's `--input PATH` replaces only that event source; silence windows always come from `/app/data/silence_windows.json` and dependency rules from `/app/data/service_dependencies.json`, resolved as absolute paths, never relative to the input or output directory. Evidence comes from `/app/incident/export_dossier.md` and the frozen snapshot at `/app/workflow/.export_report.original` — never alter the frozen file; the pre-repair SHA-256 in the audit is read from it. Neither the CLI nor the repaired workflow may read, import, or reference `/tests`, `/solution`, verifier fixtures, or expected-output artifacts.

`/app/docs/report_spec.json` is the output contract — CLI shapes, exact key sets for every report including `input_stats`, `verified_summary`, `output_paths`, and `repair_audit.json`, digest payload encodings, checksum serialization, and pinned worked examples; read it in full, since exact key names matter. How the export actually behaves is not in the contract: normalization, dedupe and its tie-breaks, silence scopes and pressure, dependency matching, lineage, causal burst, blast radius, and the flagged ordering were settled during the service review and live in the dossier's ticketed decision notes. Most of the dossier is noise, and positions were revisited more than once — early triage proposals were reversed and some interim decisions were revised again — so on every point, follow the latest decision.

Finish by running `python3 /app/log_audit.py repair --output-dir /app/output`; validation in a temporary directory does not satisfy the final handoff.
